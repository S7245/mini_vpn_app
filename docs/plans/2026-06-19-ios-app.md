# iOS App Implementation Plan

- 日期：2026-06-19
- 状态：草拟（待 grill/确认 → 转执行）
- 关联：[ADR GUI 栈=C](../adr/2026-06-19-gui-stack-c-shared-rust-core.md)、[设计 spec](../specs/2026-06-12-frontend-contract-first-macos-design.md)、[macOS plan](2026-06-12-macos-app.md)

> **iOS 是这盘最便宜的一步**：与 macOS 同属 Apple/SwiftUI 栈 → 不触发 §7.1 栈决策、**不需要 FFI**，
> 直接复用现成的 Swift `MiniVPNCore`（models/services/view-models/状态机/mock 后端全是平台无关纯逻辑）。
> 与 ADR-C 不冲突：C 的"共享层"对 Apple 端就是这套 Swift Core；Rust core 是为 Android/Windows 跨语言
> 共享才引入，由 Android 落地驱动，iOS 现在直接吃 Swift Core。

## 0. 需求整理（locked scope）

| 维度 | 决定 |
|---|---|
| 鉴权 UI | **做** login/register（走现有 mock 鉴权） |
| 真 VPN（NEPacketTunnelProvider） | **Mock-first，延后 NE**（同 macOS；NE 需 Apple entitlement，账号侧门槛） |
| 设备目标 | **iPhone 优先**（竖屏为主；iPad 适配留后续，SwiftUI 同套未来顺带） |
| 设计 | **在线出 iOS 设计稿**（HIG，逐屏 mockup → 过目 → 实现） |
| 后端 | 现有 `contracts/mock`（② 已 mock；① local-control 仍 mock） |

### 屏幕清单 + 逐屏需求（从 `contracts/` 字段推导）

1. **Auth（新）**：login（email+password）/ register；错误展示；成功→主界面。
   - 数据：`TokenPair{accessToken,refreshToken,tokenType,expiresIn}`（mock login 返回）。
   - 新增 `AuthViewModel`（login/register/logout + 会话态 isAuthenticated + 内存 token 持有）。
     这也给 macOS 当时延后的"token seam"落了第一个最小实现（内存版，真实现接 ② 时换 keychain）。
2. **Connection**：连接开关（connect/auto/disconnect）+ 状态（disconnected→connecting→connected→error）
   + 实时流量（上下行速率/累计字节）。复用 `ConnectionViewModel`（零改）。
3. **Nodes**：节点列表（shared: region/city/latency/load/tier；dedicated: staticIp/label/expiresAt）
   + 手动选 + 自动优选（selectBest）。复用 `NodeListViewModel`（零改）。
4. **Logs**：实时日志流（level/message/ts）。复用 `ConnectionViewModel.logs`。
5. **Account/Settings**：订阅（plan/status/expiresAt/deviceLimit）+ 设备列表（device_limit 衍生）+ 登出。
   复用 `AccountViewModel`（零改）。

YAGNI：不做支付/购买（契约只读 + not-implemented 占位，同 macOS）；不做 iPad 专属布局；不做真 NE。

## 1. 架构与复用

### 复用映射

| 层 | 来源 | iOS 怎么用 |
|---|---|---|
| Models / Services（含 Mock）/ ViewModels / Support / Mocks | `MiniVPNCore` | **直接复用**（加 iOS 平台即可） |
| Views（ConnectionView 等） | `MiniVPNCore`（macOS 风味） | **不复用**，iOS 走新设计（VM 不变） |
| App 外壳（MenuBarExtra/WindowGroup） | `macos-app/App` | 不复用，iOS 新外壳 |

新增：`AuthViewModel`（唯一新 VM）+ 全套 iOS Views + iOS app 外壳。

### 共享结构决定

`MiniVPNCore` 现在物理上在 `macos-app/Core`，但它本质是 **Apple 共享层**，不属 macOS。
**Phase 1 把它移到中性位置** `apple-core/`（一次性 `git mv` + 更新 macOS App 的 `project.yml` 路径），
macOS 与 iOS 两个 app 都依赖同一个包。低风险、一次到位，避免 `ios-app/` 反向依赖 `macos-app/` 路径。

目标布局：
```
apple-core/            ← 共享 Swift 包（原 macos-app/Core，加 .iOS 平台）
macos-app/App/         ← macOS 外壳（依赖 ../apple-core）
ios-app/               ← iOS app（外壳 + iOS Views，依赖 ../apple-core）
```

## 2. 阶段（按你的流程：设计 → 按模块实现 → 外壳）

### Phase 1 — Share-enable the core（结构就绪）
- `git mv macos-app/Core apple-core`；更新 `macos-app/App/project.yml` 的 package path。
- `apple-core/Package.swift`：`platforms: [.macOS(.v13), .iOS(.v16)]`。
- 用 `#if os(macOS)` / `#if canImport(AppKit)` guard macOS-only 处（`SnapshotTests` 的 `NSHostingView`；
  审计 `MockControlService` 是否有 AppKit 误引）。
- 验收：`swift test`（macOS）仍全绿；`swift build -destination 'generic/platform=iOS'` 通过；macOS App 仍 `BUILD SUCCEEDED`。

### Phase 2 — Claude design 设计 UI（gate，先于实现）
- 我逐屏出 iOS hi-fi mockup（HIG：底 TabView、NavigationStack、触控、动态字体、深浅色）：
  Auth(login/register) · Connection · Nodes · Logs · Account。
- 含：连接态主视觉（大开关 + 状态色 + 实时流量动效）、节点行布局、日志密度、表单样式。
- **过目确认后才进 Phase 3。** 设计稿存 `docs/design/ios/`（截图/说明）。

### Phase 3 — 按模块实现（TDD，subagent 一模块一任务，任务间我校验）
顺序（先骨架后填充）：
- **M1 Auth**：`AuthViewModel`（失败测试先行：login 成功置 isAuthenticated、登出清态、错误浮现）
  + Login/Register iOS Views。
- **M2 Root/Nav + 会话 gate**：未登录→Auth；已登录→TabView（Connect/Nodes/Logs/Account）。
- **M3 Connection + Traffic**：iOS ConnectionView（复用 `ConnectionViewModel`）+ 实时流量。
- **M4 Nodes**：iOS NodeListView（复用 `NodeListViewModel`）+ auto-select。
- **M5 Logs**：iOS LogsView。
- **M6 Account**：iOS Account/Settings（复用 `AccountViewModel`）+ 登出回 Auth。
每模块：失败测试 → 实现 → `swift test` 绿 → commit。新 VM（AuthViewModel）单测；复用 VM 已有覆盖。

### Phase 4 — iOS app 外壳 + 冒烟
- `ios-app/`：xcodegen iOS app target（`@main`、根视图、Info.plist、依赖 `apple-core`）。
- 注入 mock：`MockBackendService()` / `MockControlService()`（与 macOS 同一行切换点）。
- `xcodebuild -destination 'platform=iOS Simulator,name=...'` 冒烟构建（有 iOS sim 则跑）。

### 延后（不在本里程碑）
- iOS `NEPacketTunnelProvider` 扩展 + 真隧道（需 Apple NetworkExtension entitlement + provisioning——账号侧）。
- iPad 自适应布局；第三方登录；支付。

## 3. 测试与一致性
- 共享 Core 逻辑：已被 `swift test` 覆盖（24 tests）；加 iOS 平台后两端都要绿。
- 新增 `AuthViewModel` 单测（mock 驱动）。
- iOS 快照：沿用 macOS 策略——纯函数/VM 确定性测进默认套件；图像快照 env 开关（`SNAPSHOT=1`），默认 SKIP，保 gate 可信。
- iOS app 外壳：`xcodebuild` iOS-sim 冒烟（无 sim 则如实记跳过）。

## 4. 约束（同既有）
- 仅在 `mini_vpn_app-macos` worktree 操作；提交 `feat/macos-app`，push `origin/feat/macos-app`，绝不 push main；core 仓零触碰。
- `contracts/` 权威，fixture 由 `scripts/sync-mocks.sh` 同步，不手改。
- subagent 一任务一执行，任务间跑 `swift test` + 看 commit 再放行；每 stage 末 `/code-review`。

## 5. 待定/开放
- 共享包命名：`apple-core` vs `shared-swift`（默认 `apple-core`）。
- AuthViewModel 的 token 持有：本期内存版；真 ② 落地时换 Keychain + 注入 `Authorization`（呼应 macOS 延后项）。
- iOS deployment target：默认 `.iOS(.v16)`（SwiftUI 成熟度 vs 覆盖率）。
